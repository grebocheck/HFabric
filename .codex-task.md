# Task batch: idle-silence squelch + persistent voice settings + live-edit UX + recommended preset

User feedback after testing DTLN live: (1) noise is still audible while idle —
the RVC/NSF synthesizer hums and breathes even when the input is silence, so
input cleanup alone can never fix idle noise; (2) voice settings must persist
across restarts; (3) it should be clear what can be changed during a live
session; (4) ship tuned defaults for this exact setup so it sounds right out
of the box. One coherent patch. Your sandbox cannot write `.git` — do NOT
commit; leave the tree dirty for review.

## A. Idle squelch — skip conversion on silence (the actual idle-noise fix)

In `ChunkProcessor.process` (realtime only), right after the per-chunk DTLN
step and BEFORE conversion:

- Compute the denoised chunk RMS in dBFS. New engine settings:
  `silence_threshold_db` (default −48, clamp −90…−20; −90 = squelch off) and
  `silence_hold_ms` (default 400, clamp 0…2000).
- Hysteresis + hangover: squelch OPENS (voice passes) immediately when RMS
  rises above `threshold + 6 dB`; it CLOSES only after RMS has stayed below
  `threshold` for `silence_hold_ms` continuously (so word tails and short
  pauses are not chopped).
- While closed: output pure zeros for the chunk and **skip the RVC conversion
  entirely** (no ContentVec/RMVPE/synth — also saves GPU), but still append
  the denoised chunk to the rolling context so the next voiced chunk has
  history; reset `_prev_tail` to zeros so the next crossfade fades in from
  silence instead of stitching to a stale tail.
- Metrics: add `squelched: bool` to the per-chunk timings/metrics snapshot so
  the UI can show a "silence" state on the meters (and the user can verify
  the squelch is engaging).
- Offline conversion: unchanged (whole-file processing has no idle problem).
- Tests: pure-logic squelch state machine (open on loud, stays open through a
  sub-hold gap, closes after hold, reopens on voice) — extract it as a small
  class in `dsp.py` (`SquelchGate`) so it is unit-testable without audio.

## B. Persistent voice settings

- Persist the engine's `settings_payload()` to `data/voice-settings.json` on
  every successful `update_settings` (atomic write: tmp + replace; debounce
  not required — writes are tiny). Load it in `VoiceEngine.__init__` over the
  config defaults (unknown keys ignored, every value passed through the
  existing clamps; corrupt/missing file → defaults, log once).
- Device ids are indexes that may shift between boots: persist them anyway,
  and in `/status` flag `device_missing: true` inside settings when a
  persisted input/output/monitor id is not present in the current device
  list — the UI shows a small amber "re-pick device" hint on the affected
  select instead of failing.
- Tests: roundtrip (update → file → fresh engine loads it), corrupt file →
  defaults, clamping applied on load, missing-device flag set.

## C. Live-edit clarity in the UI

These already apply per-chunk (engine attributes read each chunk): pitch,
index_ratio, protect, formant, denoise, HPF, gate, gains, pass_through, and
the new squelch settings. These are stream-bound and need a session restart:
input/output/monitor device, sample rate, chunk size. In `VoicePanel`:

- Tag the stream-bound controls with a small "restart to apply" hint that
  appears ONLY while live and the value differs from the session's active
  value (track the values the session started with — return them in
  `/status` as `session_config` from the realtime module).
- Add a compact "Restart session" button next to the Stop button (stop +
  start with current settings, busy state) shown only while live.
- Everything else keeps the existing auto-apply flow (no extra apply clicks).

## D. Recommended preset for this setup

- Config defaults change: `voice_index_ratio` 1.0 → **0.75**.
- A "Recommended" chip in the Tuning header applies in one click:
  denoise=dtln, hpf=80, gate −90 (off — squelch replaces it),
  silence_threshold −48, hold 400 ms, index_ratio 0.75, protect 0.5,
  chunk 133, crossfade 0.05, extra 2.0, sample rate 48000. (Pitch is
  voice-specific — leave untouched; note that in the chip tooltip/copy.)
- The preset goes through the normal settings update → persisted by B.
